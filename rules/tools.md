# Tools & Research Rules

## Documentation lookup
**Applies to**: All work
**Rule**: Prefer Context7 for package/library/framework documentation when available.

## Web search
**Applies to**: When Context7 is insufficient or freshness matters
**Rule**: Use web search and/or the Perplexity MCP tools.

## Diagrams
**Applies to**: System design / architecture diagrams
**Rule**: Use the Nano Banana Pro MCP server.

## Public GitHub search
**Applies to**: Searching code/docs across public repos
**Rule**: Prefer the `grep-mcp` MCP server.

## Browser-based testing
**Applies to**: Inspecting frontend errors and browser testing
**Rule**: Prefer the `chrome-devtools` MCP server.

## General web search
**Applies to**: General search queries
**Rule**: Prefer the `brave-search` MCP tool/server.

## Local editing/search tools
**Applies to**: Editing files
**Rule**: Prefer `morph-mcp` `edit_file` over full-file rewrites when possible.

**Applies to**: Codebase exploration
**Rule**: Use `warpgrep_codebase_search` for broad/semantic discovery.

**Applies to**: Semantic search
**Rule**: Use `mgrep` for intent/feature discovery and onboarding; use grep/ripgrep-style tools for exact matches.
